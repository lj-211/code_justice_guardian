# 案例学习笔记：舒某、王某破坏计算机信息系统及关联犯罪案

## 一、案例基本信息

| 项目 | 内容 |
|------|------|
| **案名** | 舒某、王某破坏计算机信息系统及关联犯罪案 |
| **审理时间** | 2020年案发，具体裁判时间未明确（典型意义发布于2023年2月27日） |
| **被告人** | 舒某（主犯）、王某（技术辅助人员） |
| **涉及技术** | AI深度合成、人脸识别突破、刷机改机 |
| **核心平台** | "借贷宝"APP（网络借贷平台） |
| **刑罚结果** | 舒某：数罪并罚有期徒刑七年；王某：有期徒刑八个月 |
| **案件来源** | 人民法院依法惩治侵害公民个人信息及关联犯罪典型案例 |

## 二、核心案情与行为链条

### 2.1 主体分工

| 被告人 | 角色定位 | 具体行为 |
|--------|---------|---------|
| **舒某** | 犯罪实行人、技术使用者 | 合成虚拟视频、实施账户侵入、财产侵害 |
| **王某** | 技术辅助人、工具提供者 | 提供刷机服务、出售改装手机（明知用途） |

### 2.2 犯罪行为时间线与逻辑链

**第一阶段：技术准备（AI深度合成）**
- **技术手段**：将非法获取的被害人**静态肖像照片**，通过AI技术合成为**动态虚拟肖像视频**
- **技术目的**：生成能够通过人脸识别活体检测的动态视频

**第二阶段：设备准备（刷机改机）**
- **硬件改装**：对手机进行刷机处理，植入虚拟视频刷机包程序
- **功能实现**：使手机可调用合成的虚拟视频替代真实摄像头采集的人脸图像

**第三阶段：系统侵入（突破人脸识别）**
- **攻击目标**："借贷宝"APP的人脸识别认证系统
- **侵入方式**：使用改装手机登录账户，通过人脸识别验证（系统误识为真人）
- **侵入规模**：成功侵入他人账户**70个**

**第四阶段：数据破坏与财产侵害（多重行为）**
- **数据操作**：
  - 变更账户交易密码（控制账户）
  - **删除**债权债务记录信息（破坏数据）
  - **虚增**债权债务记录（增加虚假数据）
- **财产侵害**：
  - 直接窃取账户资金
  - 通过虚假债权债务操作转移资产
- **违法所得**：共计20余万元

**第五阶段：关联犯罪（外围行为）**
- 向他人提供用于突破人脸识别技术的手机（帮助行为）
- 伪造居民身份证件（身份伪装）
- 实施盗窃、诈骗（传统财产犯罪）

### 2.3 王某的行为定性
- **主观明知**：明知舒某购买虚拟视频刷机包程序工具系用于**突破人脸识别系统，侵入他人账户**
- **客观行为**：提供刷机服务、出售经刷机后的手机
- **罪名认定**：提供侵入、非法控制计算机信息系统的程序、工具罪

## 三、法律适用与罪名解析

### 3.1 罪名体系与数罪并罚逻辑

舒某被认定构成**五个独立罪名**，实行数罪并罚：

| 罪名 | 适用行为 | 构成要件符合性分析 |
|------|---------|-------------------|
| **破坏计算机信息系统罪** | 删除、修改、增加借贷宝账户中的债权债务记录 | 《刑法》第286条第2款：对计算机信息系统中存储的数据进行删除、修改、增加，后果严重 |
| **提供侵入、非法控制计算机信息系统的程序、工具罪** | 向他人提供用于突破人脸识别技术的手机及程序 | 《刑法》第285条第3款：提供专门用于侵入、非法控制计算机信息系统的程序、工具 |
| **伪造身份证件罪** | 伪造居民身份证件 | 《刑法》第280条第3款：伪造居民身份证 |
| **盗窃罪** | 窃取账户资金 | 《刑法》第264条：秘密窃取公私财物 |
| **诈骗罪** | 通过虚增债权债务骗取财物 | 《刑法》第266条：虚构事实、隐瞒真相骗取财物 |

**数罪并罚的正当性基础**：
- **行为独立性**：各罪名对应的行为相互独立，不存在牵连或竞合关系
- **法益侵害多元性**：分别侵害了计算机系统安全、社会管理秩序（身份证件管理）、财产权等不同法益
- **手段与目的的区分**：虽然破坏计算机信息系统是手段，盗窃、诈骗是目的，但因破坏计算机信息系统罪已被独立评价为"后果严重"，且手段行为超出了一般盗窃、诈骗的通常手段范畴，故实行并罚而非择一重罪

### 3.2 核心罪名：破坏计算机信息系统罪的认定要点

**适用条款**：《刑法》第286条第2款（数据操作型）

**关键构成要件解析**：

1. **"违反国家规定"**：
   - 包括《网络安全法》《个人信息保护法》《数据安全法》等关于数据安全和系统安全的规定
   - 未经授权对他人账户数据进行操作即属违反

2. **"对计算机信息系统中存储的数据进行删除、修改、增加"**：
   - **删除债权债务记录**：导致平台数据完整性被破坏，可能影响后续债务追偿
   - **虚增债权债务记录**：增加虚假数据，扰乱平台数据真实性
   - **变更交易密码**：属于对账户控制权限数据的修改

3. **"后果严重"的认定**：
   - 侵入账户70个（符合《解释》第4条第（四）项"破坏计算机信息系统功能、数据或者应用程序，造成其他严重后果"）
   - 违法所得20余万元（符合《解释》第4条规定的“后果特别严重”的情形）
   - 涉及金融借贷系统，危害金融管理秩序

### 3.3 人脸识别突破行为的法律定性

**技术本质**：通过AI深度伪造（Deepfake）技术，将静态照片转化为动态视频，欺骗人脸识别系统的活体检测机制。

**法律评价**：
- **不属于"技术手段"的合法抗辩**：舒某辩称可能涉及"使用技术手段"，但法院认定该技术手段**突破安全认证机制**，具有非法性
- **对"身份认证数据"的侵害**：人脸信息属于《个人信息保护法》第28条规定的**敏感个人信息**，突破人脸识别即是对身份认证数据的非法利用
- **与非法获取计算机信息系统数据罪的区分**：
  - 若仅突破人脸识别后**查看**数据，可能构成非法获取计算机信息系统数据罪（第285条第2款）
  - 但本案中存在**删除、修改、增加**操作，故升级为破坏计算机信息系统罪（第286条第2款）

## 四、争议焦点与疑难问题

### 4.1 AI合成视频是否属于"程序、工具"？

**问题**：舒某合成的虚拟肖像视频，以及刷机包中的程序，是否属于《刑法》第285条第3款规定的"专门用于侵入、非法控制计算机信息系统的程序、工具"？

**分析**：
- **程序、工具的定义**：根据《最高人民法院、最高人民检察院关于办理危害计算机信息系统安全刑事案件应用法律若干问题的解释》第2条，是指具有避开或者突破计算机信息系统安全保护措施，未经授权或者超越授权获取计算机信息系统数据的功能的程序、工具
- **AI合成视频的性质**：视频本身属于**数据**，但生成和调用视频的程序（刷机包程序）属于**工具**
- **法院立场**：刷机包程序通过欺骗人脸识别系统，实现了"避开安全保护措施"的功能，应认定为"侵入工具"

### 4.2 刷机手机的性质认定

**争议**：王某出售的"经刷机处理的手机"是硬件设备，能否认定为"程序、工具"？

**实务观点**：
- **整体性功能评价**：虽然手机是硬件，但经刷机植入特定程序后，其整体成为**实施侵入的专用设备**
- **司法解释扩张**：司法实践中，将**含有特定犯罪软件的硬件设备**整体认定为"工具"，符合打击此类犯罪的需要
- **明知的证明**：王某"明知舒某用于突破人脸识别系统"的主观故意，是定罪关键

### 4.3 破坏计算机信息系统罪与盗窃罪的界限

**关键区分**：
- **行为指向**：破坏计算机信息系统罪指向**数据本身的完整性、真实性**（删除、修改记录）；盗窃罪指向**财产权的转移**（将资金转走）
- **法益评价**：即使为了盗窃而修改数据，当修改行为本身**严重危害系统安全**时，应单独评价
- **本案特殊性**：舒某不仅转移资金，还"删除债权债务记录"、"虚增债权债务"，这些行为对平台数据秩序的破坏超出了盗窃罪所能评价的范围，故需并罚

## 五、典型意义与实务启示

### 5.1 对AI技术滥用的刑法规制

**技术风险警示**：
- **Deepfake技术的刑事风险**：将静态照片合成动态视频突破人脸识别，已从技术工具演变为犯罪工具
- **黑产链条化**：本案呈现出"照片获取→AI合成→刷机改机→账户侵入→资金窃取"的完整黑产链条

**法律评价趋势**：
- 司法机关对**利用AI技术突破安全认证**的行为采取**零容忍**态度
- 即使技术本身中立，但用于**欺骗身份认证系统**即构成对计算机信息系统安全的破坏

### 5.2 人脸识别系统的安全义务

**对平台企业的警示**（典型意义明确提示）：

1. **技术迭代义务**：企业应当高度重视人脸识别等信息技术的迭代升级，提升活体检测算法对深度伪造视频的识别能力
2. **多因子认证**：单一的人脸识别认证存在风险，应结合短信验证、设备指纹、行为特征等多因子认证
3. **异常监测**：建立对"删除债权债务记录"等高风险操作的实时监测和二次验证机制
4. **数据备份**：对债权债务等关键数据进行不可篡改的备份，防止本案中的删除操作造成实际损失

### 5.3 个人生物识别信息保护

**个人防护建议**：
- **照片泄露风险**：个人肖像照片一旦被非法获取，可能被用于AI合成突破人脸识别
- **减少暴露**：在社交媒体减少高清正面照片的公开传播
- **授权谨慎**：对APP的人脸识别授权保持警惕，了解其数据存储和安全保护措施

### 5.4 技术辅助人员的刑事责任边界（王某案警示）

**"明知"的认定标准**：
- 王某明确知道舒某购买工具系用于"突破人脸识别系统，侵入他人账户"
- 对于提供刷机、改机服务的技术人员，**只要明知用途违法，即构成共犯**

**技术提供者的合规红线**：
- 不提供用于突破安全认证的刷机包、外挂程序
- 不对客户提供明显用于违法犯罪的技术服务（如修改手机摄像头调用逻辑以欺骗APP）
- 建立客户用途审查机制，对明显异常的需求保持警惕

---

本案警示：在AI技术普及的时代，**技术能力不等于法律豁免**，利用技术手段突破安全认证、破坏数据完整性，将面临严厉的刑事制裁。
