## 一、案情梳理

### 1.1 被告人身份

| 被告人 | 身份背景 | 角色定位 |
|--------|----------|----------|
| **卫某龙** | 北京某大型网络公司前员工（2012-2014年任职） | 实行犯：具体实施登录、下载数据行为 |
| **龚某** | 该大型网络公司运营规划管理部在职员工 | 帮助犯：提供账号、密码、Token令牌 |
| **薛某东** | 卫某龙的商业合作伙伴 | 共犯：负责出售非法获取的数据 |

### 1.2 行为经过

```
2016年6月 ——→ 2016年9月
     ↓
龚某提供账号、密码、Token令牌
     ↓
卫某龙异地登录公司内部管理开发系统
     ↓
查询、下载系统中储存的电子数据
     ↓
交由薛某东通过互联网出售牟利
     ↓
违法所得：人民币 3.7万元
```

### 1.3 关键事实要点

- **龚某的权限**：因工作需要拥有登录内部管理开发系统的账号、密码、Token令牌，具有查看**工作范围内**相关数据信息的权限
- **公司禁止性规定**：禁止员工私自在内部管理开发系统查看、下载**非工作范围内**的电子数据信息
- **行为特征**：经事先合谋，内外勾结，多次在**异地登录**

## 三、裁判理由分析

### 3.1 法律适用

**《刑法》第285条第2款**（非法获取计算机信息系统数据罪）：
> 违反国家规定，侵入前款规定以外的计算机信息系统或者采用其他技术手段，获取该计算机信息系统中存储、处理或者传输的数据，情节严重的，处三年以下有期徒刑或者拘役，并处或者单处罚金；**情节特别严重的，处三年以上七年以下有期徒刑，并处罚金**。

### 3.2 "侵入"的认定标准

法院对"侵入"作出了扩张性解释：

| 情形 | 是否属于"侵入" | 本案适用 |
|------|----------------|----------|
| 未取得授权擅自进入 | ✅ 属于 | ❌ 不适用（龚某有授权） |
| **超出授权范围进入** | ✅ **属于** | ✅ **适用** |

**核心裁判观点**：
> 超出授权范围使用账号、密码、Token令牌登录系统，也属于侵入计算机信息系统的行为。

### 3.3 "情节特别严重"的认定

依据**《最高人民法院、最高人民检察院关于办理危害计算机信息系统安全刑事案件应用法律若干问题的解释》（法释〔2011〕19号）第1条**：

| 情节标准 | 本案情况 |
|----------|----------|
| 违法所得 **二万五千元以上** | 违法所得 **3.7万元** ✅ |
| 量刑档次 | **三年以上七年以下** |

## 四、裁判要旨

```
┌─────────────────────────────────────────────────────────────┐
│  非法获取计算机信息系统数据罪中"侵入"的认定要点：            │
│                                                              │
│  1. "侵入"包括两种情形：                                    │
│     • 未经授权擅自进入                                      │
│     • 超出授权范围进入 ← 本案重点                           │
│                                                              │
│  2. 行为方式不限于技术手段：                                │
│     • 技术手段侵入（如黑客攻击）                            │

│                                                              │
│  3. 定罪逻辑链条：                                          │
│     超权限登录（侵入）→ 下载数据（获取）→ 情节严重 → 构成本罪│
└─────────────────────────────────────────────────────────────┘
```

## 六、案例启示

### 6.1 对司法实践的启示

1. **"侵入"概念的扩张解释**：本案明确了"侵入"不仅包括技术手段的非法进入，还包括**权限滥用型**的非技术侵入，体现了对计算机信息系统安全的全面保护。

2. **内外勾结型犯罪的认定**：即使内部人员拥有合法账号，但将账号提供给外部人员用于非授权目的，或自身超权限使用，均可能构成"侵入"。

3. **共同犯罪的认定**：三人分工明确（提供权限→实施获取→销赃变现），形成完整的犯罪链条，均按非法获取计算机信息系统数据罪共犯处理。

### 6.2 对企业合规的启示

| 风险点 | 防范建议 |
|--------|----------|
| 员工权限管理松散 | 建立最小权限原则，定期审计账号使用记录 |
| 异地登录缺乏监控 | 设置异常登录预警（如异地、非工作时间登录） |
| 数据下载行为无追溯 | 实施数据分级分类，敏感数据下载需审批留痕 |
| Token令牌管理不善 | 定期更换令牌，禁止共享账号密码 |
