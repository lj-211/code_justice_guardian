# “小白改机”案（通过 HOOK 方式修改应用程序数据）

## 一、案件基本信息

| 项目 | 内容 |
|:---|:---|
| **案件名称** | 徐忠明破坏计算机信息系统案 |
| **案号** | （2018）沪0105刑初1034号（一审）、（2019）沪01刑终1632号（二审） |
| **审理法院** | 上海市长宁区人民法院（一审）、上海市第一中级人民法院（二审） |
| **裁判日期** | 2020年4月28日（终审裁定） |
| **法宝引证码** | CLI.C.102923777 |
| **被告人** | 徐忠明（主犯）、潘安、吴诚、张荣（从犯） |
| **罪名** | 提供侵入、非法控制计算机信息系统程序、工具罪 |

---

## 二、案件事实

2017年5月，被告人徐忠明制作了针对"饿了么"等平台使用的名为"XX"（即"小白改机"）的软件，后招募被告人潘安、吴诚、张荣等人作为代理商，在网络上进行销售获利。

**销售情况：**
- 徐忠明：销售8,000余人次，获利人民币5万余元
- 潘安：销售4,900余人次
- 吴诚：销售获利2万余元
- 张荣：销售160余人次

**技术特征：** 经鉴定，"XX"软件具有通过Hook系统API的调用结果实现修改Android系统中的设备串号、AndroidID、设备品牌、型号、设备名、系统版本等信息的功能。该软件通过Hook应用程序包名称含有"me.ele"的应用中"jaca.util.HashMap.put()"的函数，篡改了其他应用程序在获取系统信息的处理过程中的数据，具有破坏性。

---

## 三、核心争议焦点

### 1. 罪名的认定分歧

| 观点 | 主张内容 |
|:---|:---|
| **检察机关抗诉观点** | 制作及传播"XX"软件的行为应当构成**破坏计算机信息系统罪**，原判适用法律错误，导致量刑畸轻 |
| **被告人上诉观点** | "XX"软件拦截和修改用户数据得到了用户授权，不构成犯罪 |
| **辩护人观点** | "XX"软件收集数据系经用户授权修改自己手机设备信息，属对"饿了么"等平台非法调取信息的正当防卫，依法不构成犯罪 |
| **法院最终认定** | 构成**提供侵入、非法控制计算机信息系统程序、工具罪**，驳回上诉，维持原判 |

---

## 四、关键技术分析：侵入性特征的认定

### 要点一：HOOK系统API的技术路径

"小白改机"软件的技术原理：
- 通过**Hook系统API**的调用结果（如果只是单纯HOOK系统API不足以认定存在侵入性）
- 实现修改Android系统中的手机串号、Android ID、设备品牌、型号、设备名等信息
- 篡改其他应用程序在获取系统信息处理过程中的数据

### 要点二：侵入性的核心认定——"二次篡改"机制

**这是本案认定侵入工具符合技术特征的关键所在。**

法院特别指出：

> "该软件**并不是单纯的将软件使用者的手机串号、Android ID、设备品牌、型号、设备名修改后提供了前来调取信息的'饿了么'等其他应用程序**，**而是在'饿了么'等其他应用程序调取到软件使用者的手机串号、Android ID、设备品牌、型号、设备名等信息后，再将信息予以篡改**。"

### 要点三：侵入性的法律内涵

法院据此认定：

> "由此可知，该功能的核心目的在于**篡改其他应用程序在获取系统信息处理过程中数据**，既然是篡改，那就意味着**不可能得到授权，具有侵入性**。"

**关键逻辑链条：**

```
二次篡改机制 → 应用程序已读取真实信息 → 再修改返回数据 
                                    ↓
                        必然未经应用程序授权
                                    ↓
                              具有侵入性
                                    ↓
                    构成提供侵入计算机信息系统程序、工具罪
```

## 五、为何不构成破坏计算机信息系统罪

法院从三个维度进行了详细论证：

### 1. 不符合刑法第286条第1款
- 该款要求"造成计算机信息系统不能正常运行"
- 本案软件仅使"饿了么"APP收到错误返回信息，**未造成其不能正常运行**

### 2. 不符合刑法第286条第2款
- 该款主要针对**具体实施破坏行为的人**
- 软件开发及销售者目的只是牟利，与购买者无共谋，**不存在共同犯罪故意**

### 3. 不符合刑法第286条第3款（关键论证）
**技术规范vs刑法规范的区分：**
- 技术规范意义上的"破坏性程序"强调**非授权性**（依据《破坏性程序操作规范》）
- 刑法意义上的"破坏性程序"要求危害程度与**计算机病毒基本相当**

**"小白改机"与计算机病毒的差异：**

| 特征 | 计算机病毒 | 小白改机 |
|:---|:---|:---|
| 自我复制性 | 有 | 无 |
| 自动触发性 | 有 | 无 |
| 主动植入性 | 能主动植入系统 | 需用户主动运行操作 |
| 破坏程度 | 严重影响系统正常运行 | 仅篡改特定数据传输 |

**体系解释的角度：**
> "若将制售改机软件的行为均以破坏计算机信息系统罪定罪，那么刑法第285条提供侵入、非法控制计算机信息系统程序、工具罪将很大程度上被虚置。"

## 六、为何构成提供侵入、非法控制计算机信息系统程序、工具罪

### 1. 侵犯的计算机信息系统不限于特定领域
- 刑法第285条第3款不仅是第1款（国家事务等领域）的帮助行为
- 也是第2款（非法获取数据、非法控制）的帮助行为
- 故所涉领域不限于国家事务、国防建设、尖端科学技术领域

### 2. "小白改机"软件的功能具有侵入性（核心认定）

如前所述，通过"二次篡改"机制，软件**未经"饿了么"等应用程序授权**即进入其计算机信息系统，具有非正当性和侵入性。

### 3. 不属于"中性程序、工具"
- "中性程序、工具"既可以用于合法目的，也可以用于非法目的
- 不能具备在未经授权或超越授权的情况下侵入、自动获取数据或者控制他人计算机信息系统的功能
- "小白改机"专门针对"饿了么"等APP设计，**显然不是"中性程序、工具"**

## 七、案例启示与实务要点

### 1. 技术事实的查明至关重要

本案法院深入分析了软件的技术原理，特别是区分了：
- 单纯修改系统信息后再提供给应用程序（可能属于用户自主权范围）
- **在数据传输过程中进行二次篡改**（必然未经授权，具有侵入性）

### 2. "Hook"技术的刑事风险边界

| 技术行为 | 风险等级 | 理由 |
|:---|:---|:---|
| 仅修改本机系统信息 | 较低 | 属于用户对自有设备的管理 |
| Hook系统API | 中等 | 可能影响其他应用正常运行 |
| **Hook特定应用程序函数并篡改传输数据** | **高** | **直接侵入他人计算机信息系统** |

### 3. 罪名区分的核心标准

- **破坏计算机信息系统罪**：关注是否造成系统不能正常运行、是否具有自我复制/自动触发特性
- **提供侵入、非法控制计算机信息系统程序、工具罪**：关注是否**未经授权侵入**他人系统

### 4. 辩护策略的反思

本案辩护人以"用户授权"和"正当防卫"为由进行辩护，但法院指出：
- 用户只能授权修改**自己的设备信息**
- 无法授权**篡改他人应用程序获取的数据传输过程**
- "正当防卫"的辩护意见于法无据
